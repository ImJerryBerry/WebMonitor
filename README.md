# 多网站状态自动捕获监听助手

这是一个轻量级、跨平台的 Python 自动化工具。它可以同时并发监听多个网站的内容变化（例如：公告更新、招标信息、更新日志等），一旦发现网页内容发生更新，系统会立即在桌面弹出通知、播放提示音，或者发送企业微信机器人消息。

## ✨ 核心功能

- 🚀 **多线程监听**：支持同时监听多个互不干扰的网站，每个网站可独立设置刷新间隔。
- 🔔 **多方式跨平台通知**：支持 Windows、macOS、Linux 的桌面弹窗通知与系统音效，支持企业微信机器人推送通知。
- 📂 **网页快照归档**：自动将发生变动的 HTML 网页保存到本地文件夹。

## 🛠️ 安装与运行环境

1. **安装 Python**：请确保你的电脑上已安装 Python 3.9 或以上版本。
2. **下载项目**：将本项目的所有 `.py` 文件及 `requirements.txt` 下载到同一个文件夹中，或在终端中执行以下命令：
   ```bash
   git clone https://github.com/ImJerryBerry/WebMonitor.git
   ```
3. **安装依赖**：
   在终端中进入该文件夹，运行以下命令安装必备的第三方库：
   ```bash
   pip install -r requirements.txt
   ```

## ⚙️ 配置说明

在运行程序之前，你需要打开并修改 **`config.py`** 文件，配置你想要监听的网站信息。

### 1. 配置监听网站和策略

找到 `WEBSITES` 列表，你可以按照以下格式自由添加、删除或修改：

```python
WEBSITES = [
    {
        "name": "百度",                   # 自定义网站名称
        "url": "https://www.baidu.com/", # 目标网页链接
        "interval": 60,                  # 刷新间隔（秒），默认60秒
        "save_mode": 1,                  # 网页快照保存模式（详见下方说明）
        "notify_modes": [1, 2, 3]        # 通知模式（详见下方说明）
    }
]
```

关于 `save_mode` (保存模式) 的说明：

- **`0`**：不保存任何 HTML 文件到本地，只进行弹窗和声音通知。
- **`1`**：保存最新的一次 HTML，并自动清理上次记录的旧文件（推荐）。
- **`2`**：保存每一次更新的 HTML，并保留所有的历史旧文件，适用于需要严格归档留存的场景。

关于 `notify_modes` (通知策略) 的说明：

通过传入一个数字列表组合，为不同网站灵活配置：
- **`0`**：不通知（仅在控制台输出信息）
- **`1`**：播放系统音效通知
- **`2`**：弹出系统桌面通知
- **`3`**：推送给企业微信机器人通知

在终端中运行通知程序，可测试各通知方式是否能正常工作：

```bash
python notifier.py
```

### 2. 配置企业微信机器人（可选）

如果需要将通知发送到微信，请在 config.py 中填入群机器人的 Webhook 地址：

```python
WECHAT_WEBHOOK = "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY_HERE"
```

有关企业微信 Webhook 地址的使用方法，请参见[企业微信消息推送配置说明](https://developer.work.weixin.qq.com/document/path/99110)。

## 🚀 开始运行

配置好 `config.py` 后，在终端中运行主程序即可：

```bash
python main.py
```

- 程序启动后，会自动创建 `captures/` 文件夹。
- 控制台会实时打印各个网站的监听状态，如果在设定时间内网站未更新，会打印时间戳日志。

---

## 📁 项目结构

```text
├── main.py              # 程序的入口文件，负责启动多线程
├── config.py            # 配置文件（设定要监听的网址、保存模式、伪装请求头）
├── monitor.py           # 核心逻辑（抓取网页、哈希比对、保存文件、读取历史）
├── notifier.py          # 通知模块（处理跨平台的弹窗与声音报警）
├── requirements.txt     # 第三方依赖库列表
└── captures/            # 存放网页历史HTML快照的文件夹（运行时自动生成）
```
